- **Fix: Sync client target choices with installed plugins**: `jac build` and `jac start` now derive `--client` choices from the registered target list so plugin-provided targets stay visible and selectable.
- **Fix: Restore `@jac/mobui` web bundler wiring**: The web Vite build now emits `compiled/client_mobui.js` and aliases the bare `@jac/mobui` specifier (plus `react-native` to `react-native-web` when the project declares the `react-native-web` dependency) so mobUI apps lower to the DOM and build on the web target, not just native. `useWindowDimensions` is also re-exported from the `@jac/mobui` primitive surface. Previously the web path silently dropped these, so every mobUI example failed `jac build --client web` with an unresolved `@jac/mobui` import.
