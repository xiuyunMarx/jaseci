- **Refactor: jac-scale reorganized into a capability-modular package**: every concern now lives in one folder with its port, adapters, factory, and impl annexes - `deploy/` (target/database/autoscale), `observability/`, `emailer/`, `identity/`, `persistence/`, `memory/`, `server/`, `webhook/`, `config/`, `models/`, `shared/`, plus the `runtime/` serving layer with live `Gateway` (ProcessGateway/EmbeddedGateway) and `ServiceTransport` (Http/InProcess) ports selected by the realization planner. The old grab-bag `abstractions/`, `factories/`, `targets/`, `utilities/`, `providers/`, and `microservices/` folders are gone, and the package root holds only `plugin.jac`. Microservice routes are served at a bare `/<service>` (no `/api/` prefix): a service owns only its member subpaths (`/<service>/walker|function|...`) so a same-named SPA route still falls through to the static client.
