- **Fix: deterministic `jac gen-jir-registry` output**: The registry generator now runs its `jac format` step from a neutral working directory, so the generated `jir_registry.jac` is byte-identical regardless of the invoking project's `[check.lint]` settings. Previously, a project that enabled the opt-in deslop rules (`strip-comments`/`strip-docstrings`) would strip the docstrings from this generated file, breaking the `gen-jir-registry --verify` CI gate.
