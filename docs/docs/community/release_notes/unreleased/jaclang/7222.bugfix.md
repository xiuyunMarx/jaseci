- **Fix: Lazy singletons initialized from an `Any` value no longer mis-report their return**: The `if slot is None { slot = init(); } return slot` singleton idiom over an optional `T | None` global or attribute rejected the `return` with a spurious `Cannot return T | NoneType, expected T` (E1002) whenever `init()` had an imprecise `Any` type, such as `json.loads` or a call into an installed-but-untyped third-party package that degrades to foreign `Any`. The assignment-based narrowing engine now recognizes that a flat assignment of an imprecise value still writes a concrete non-`None` value and narrows the optional slot to its non-`None` form, instead of discarding the narrow and letting the declared union cascade into a misleading diagnostic. A genuinely incompatible concrete initializer is unaffected and is still reported at the assignment site.
