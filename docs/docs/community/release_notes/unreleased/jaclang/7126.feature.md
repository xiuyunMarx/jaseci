- **Checker: WHATWG browser host APIs modeled for cl code**: `jac check` no longer false-positives on standard browser code in client modules. The JS type environment now models `fetch` → `Response` (`ok`, `status`, `json()`, `text()`, `body`), `Headers`, `ReadableStream`/`ReadableStreamDefaultReader` (`read()` → `{done, value}`), `TextDecoder`/`TextEncoder`, a typed `navigator.clipboard` (`writeText`/`readText`), and the document pointer-lock surface (`pointerLockElement`/`exitPointerLock()`). `await` now unwraps `Promise[T]` to `T` (via `Promise.__await__`), so awaited results no longer cascade E1055/E1001. Dynamic payloads (`resp.json()`, `JSON.parse`) intentionally stay `object`, re-type them at the boundary with an `as` cast. Also fixes two evaluator bugs surfaced by the same QA: every `round(...)` call failed E1054 (typeshed's private `_SupportsRound1/2` protocols were missing from the structural-protocol table), and `{**spread}` dict literals inferred the spread dict as one entry value instead of merging its key/value types.
- **`jac precommit` correctness + migration**: `jac precommit --staged --verify` now gates the *staged blobs* (via a temp index checkout when they differ from the worktree) instead of the working-tree copies, so unformatted staged content can no longer slip into a commit past a clean worktree, and clean staged content is no longer rejected over a dirty worktree (partial staging works). Fix mode also fails with a re-stage hint when the index still holds different bytes than the (clean) worktree. `jac precommit --install` now migrates automatically over a hook generated by the python pre-commit framework (backing it up as `pre-commit.bak`), and a new `--force` flag replaces any other existing hook, also with a backup. A new `[precommit] typecheck = false` jac.toml knob limits the gate to formatting/lint for codebases that are not yet `jac check`-clean (this repo sets it).
- **Dev-source precedence: a repo's own `[dev] jaclang_source` beats a linked binary's baked link**: a dev binary built with `zig build -Ddev` from checkout A, run inside checkout B whose jac.toml declares `[dev] jaclang_source`, now compiles with B's own tree instead of silently using A's. The baked link remains the fallback wherever no stanza is in scope, and `JAC_NO_DEV_SOURCE=1` still suppresses only the toml loop (a linked binary keeps its baked compiler, since it ships no bundled one).
