- **Bugfix: `jac test` again runs every `.jac` file under the configured `[test] directory`** (issue #7151): the pytest-based runner only collected `test_*.jac` / `*.test.jac`, so a project with a configured `[test] directory` (or `jac test -d <dir>`) silently collected 0 items from suites whose files followed neither naming convention (a regression from the native runner). `jac test` now passes the declared test root to the pytest plugin via a new `--jac-test-dir` option, and every `.jac` file beneath that root is collected as a suite (still excluding `impl` annexes and `fixtures/` inputs). Positional path targets keep pytest naming rules, so `jac test some/dir` and plain `pytest` are unchanged.
