- **Config: single file-anchored `jac.toml` resolver**: Project-root resolution is now one canonical walk that every call site delegates to, and `jac format`/`jac check` lint config is resolved from the file being processed (nearest `jac.toml`, no ancestor merge) rather than the current working directory, so a file's rules no longer depend on where the command is run from, and an opinionated repo config (e.g. the deslop rules) never leaks onto files outside any project. Exclude-pattern matching is also unified into one shared helper with depth-agnostic path patterns.
