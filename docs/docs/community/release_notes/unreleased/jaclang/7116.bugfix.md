- **Fix: microservice gateway no longer crash-loops on the `__guest__` user creation race**: at boot every pod ensures the built-in guest user exists via a check-then-create against the shared MongoDB. With several pods starting together the checks all miss, all insert, and the losers hit `E11000 duplicate key ... identities.value_normalized: "__guest__"`, which propagated out of `_store_user` uncaught and crash-looped the gateway. The Mongo `_store_user` now translates a duplicate-key insert into the existing `IdentityTakenError`, and `create_user` returns the normal `IDENTITY_TAKEN` result, so the concurrent create is idempotent (the guest-ensure re-fetches the now-existing user) instead of fatal.
- **Fix: `dev`-channel binary download no longer hits the GitHub API rate limit**: `download_release_binary` resolved the release asset through the anonymous GitHub API (60 requests/hour per IP), so busy runners and repeated deploys failed with `HTTP 403: rate limit exceeded`. The rolling `dev` prerelease has a fixed tag and fixed asset names, so the download now builds the `releases/download/dev/jac-dev-linux-<arch>` URL directly and skips the API entirely (that host isn't API-rate-limited). Stable releases still resolve their versioned asset through the API.
- **Fix: microservice gateway no longer crash-loops building the admin console on-pod**: in dev source-overlay deploys the shipped overlay carries no pre-built admin `_dist`, so each gateway pod built the ops console from source at boot - a heavy `bun` build that could get the container OOM-killed under a busy node, tripping the restart-based crash detector before the gateway ever bound its port. The admin console is now built once on the host during overlay packing (as the client bundle already is) and shipped pre-built, so the pod copies the finished dist and starts immediately; it falls back to the on-pod build (with a warning) only if the host build cannot run.
