- **Feature: Forbid raw HTML host tags in mobUI projects (E1105)**: Setting `client_kind = "mobui"` under `[project]` in `jac.toml` marks a project as targeting React Native as well as the web. In a mobUI project, raw HTML intrinsic tags in JSX (`<div>`, `<span>`, `<button>`, ...) now raise `E1105` with a fix-it pointing at the `@jac/mobui` / React Native component to use instead (`div` → `View`, `span` → `Text`, `button` → `Pressable`, `img` → `Image`, ...). Uppercase user components are unaffected. Enforcement runs in the IR-generation schedule, so it fires on both `jac check` and `jac build`/dev. The client kind and project root are discovered from each module's own path (its project's `jac.toml`), never the process cwd; modules outside the project root and `.cl.jac` web-boundary files (which always target the browser) are exempt.
- **Enhancement: Scope-aware JSX host-tag detection (E1105 / W1050)**: The mobUI-project guard now uses symbol-table scope resolution rather than a lexical case check. Imported lowercase components (`<myWidget>`) are recognised as in-scope user components and are no longer flagged with `E1105`. The type evaluator is updated in parallel so that in-scope lowercase components in mobUI projects also receive correct prop-type validation without spurious `W1050` ("Unknown intrinsic JSX element"). Web-project behaviour is unchanged.
