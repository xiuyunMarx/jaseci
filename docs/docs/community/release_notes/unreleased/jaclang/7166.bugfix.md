- **Fix: persisted objects could lose enum/declared field types on reload when the defining module was not the ambient `__main__`**: field-type resolution for deserialization coercion ran lazily via `get_type_hints`, which resolves a class's lazy string annotations through `sys.modules[cls.__module__]`. If that slot no longer held the defining module at reload time (e.g. another entrypoint module imported as `__main__` on the same test worker or server process), resolution failed, the unresolved string annotations were cached permanently, and coercion silently degraded (enum fields came back as raw strings). Field types are now resolved eagerly at class registration, while the defining module is guaranteed live, and a failed lazy resolution is no longer cached, so a later call can still recover.
